Mercedes-Benz India Digital Assembly Platform

User Manual & Technical Documentation

Date: December 18, 2025
Version: 1.0.0
Confidentiality: Internal Use Only

1. Introduction

This document serves as the comprehensive guide for the Mercedes-Benz India Digital Assembly Platform. The system is designed to provide real-time visibility into production metrics, manage operational shifts, and secure sensitive audit logs.

2. System Access

Login Screen

The entry point to the application is secured via a dedicated Login interface. Users must authenticate with their registered Email and Password.

  • Admin Access: Can view all modules.
  • User Access: Restricted to dashboard and andon screens.

3. Dashboard

Upon successful login, users are greeted with the main Dashboard. This screen aggregates critical data from across the factory floor.

  • Top Charts: Display alarm counts categorized by stakeholder (Logistics, Quality, etc.).
  • Interactive Elements: Hover over data points to see precise values and timestamps.

4. Navigation System

Sidebar Menu

The application features a collapsible sidebar with a premium video background. It categorizes functions into:

  • Andon: Production line monitors.
  • Breakdown: Incident reporting.
  • Management: Administrative configuration.
  • Reports: Historical data and audits.
  • Authorization: (Admin Only) User management.

5. Operational Screens (Andon)

The Andon screens provide high-visibility metrics for the shop floor.

Trim 4 Overview

Displays real-time target vs. actuals, takt time, and live status.

6. Administration & Security

Audit Trail

Restricted Area: Only accessible to Administrators.

The Audit Trail records every interaction within the system. It features:

  • Timestamped logs.
  • User identiy verification.
  • Action categorization (Login, Navigate, Update).
  • Excel Export: One-click download of reports.

Access Matrix

Allows admins to control granular permissions for every user in the system.

7. Technical Details

  • Framework: React.js
  • Architecture: Component-based with localized Context state.
  • Auto-Save: Logs are automatically archived to the server every hour.
